Build a new feature module called **Client Portal – CP‑1: Authentication + Dashboard**.  
This module must implement client login, session handling, role-based access, and the main Client Dashboard.  
Implement all functionality and complete work through test.

------------------------------------------------------------
### 1. Client Authentication

#### 1.1 Magic Link Login
- Implement magic link authentication for clients.
- Use email-based login only (no password).
- Store client sessions securely.
- Redirect authenticated clients to the Client Dashboard.

#### 1.2 Role-Based Access
- Enforce `role = client`.
- Prevent access to Awo Portal, House Portal, Admin Portal.
- Redirect unauthorized users to login.

#### 1.3 Session Handling
- Persist client session across page reloads.
- Auto-logout on token expiration.
- Provide “Logout” button.

#### 1.4 Registration Flow
- Allow new clients to register:
  - name
  - email
  - phone
  - timezone
- Validate email uniqueness.
- Create record in `clients` table.

------------------------------------------------------------
### 2. Client Dashboard

#### 2.1 Dashboard Overview
Display:
- Next upcoming consultation (if any)
- Pending booking requests
- Pending payments
- Recent consultations
- Ebo payment status (if applicable)

#### 2.2 Quick Actions
Buttons:
- “Book Consultation”
- “View Consultation History”
- “View Payments”
- “Update Profile”
- “Messages” (Phase 1 stub)

#### 2.3 Dashboard Cards
Implement cards for:
- Upcoming Consultation
- Pending Payments
- Recent Consultations
- Booking Requests

Each card must:
- Be clickable
- Navigate to the correct Client Portal module

------------------------------------------------------------
