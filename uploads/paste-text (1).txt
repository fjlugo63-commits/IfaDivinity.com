Project Name: Ifa Divinity Marketplace (ifa-staging / ifa-prod)
Template: Marketplace / E‑commerce

Goal: Scaffold a secure MVP marketplace for Ifa divination products and services (physical ritual items and service bookings such as Egbo). Generate frontend (Next.js), serverless functions, Supabase DB + RLS SQL, Stripe checkout + webhook edge functions, GitHub Actions CI with staging preview and manual production gate, Supabase Storage signed uploads, and seed data for staging.

Core Flows:
- Buyer: browse, search, filter, product detail, add to cart, checkout (Stripe Checkout), order confirmation email.
- Seller: sign up, profile, create/manage product and service listings (images, price, inventory, shipping, service_type e.g., egbo), view orders, availability calendar for services.
- Admin: manage products, orders, users, seller verification, refunds, content moderation.
- Services: booking flow for readings and Egbo rituals with calendar availability and optional digital delivery.

Data Model (create these tables and indexes; include RLS policies):
- users: id uuid pk default gen_random_uuid(), email text unique not null, role text default 'buyer', name text, created_at timestamptz default now()
- sellers: id uuid pk default gen_random_uuid(), user_id uuid references users(id), display_name text, bio text, verified boolean default false, created_at timestamptz default now()
- products: id uuid pk default gen_random_uuid(), seller_id uuid references sellers(id), title text not null, description text, price_cents integer not null, currency text default 'USD', inventory integer default 0, images text[], category text, service_type text, duration_minutes integer, location_type text default 'remote', published boolean default false, created_at timestamptz default now()
- services (optional): same fields as products but optimized for bookings (if you prefer separate table)
- orders: id uuid pk default gen_random_uuid(), buyer_id uuid references users(id), total_cents integer not null, currency text default 'USD', status text default 'pending', stripe_payment_id text, created_at timestamptz default now()
- order_items: id uuid pk default gen_random_uuid(), order_id uuid references orders(id) on delete cascade, product_id uuid references products(id), quantity integer not null, unit_price_cents integer not null
- bookings: id uuid pk default gen_random_uuid(), seller_id uuid references sellers(id), buyer_id uuid references users(id), product_id uuid references products(id), start_time timestamptz, end_time timestamptz, status text default 'scheduled', meeting_link text
- audit_logs: id uuid pk default gen_random_uuid(), actor_id uuid, action text, resource text, metadata jsonb, created_at timestamptz default now()
- Indexes: products(seller_id), products(category)

RLS Policies (apply and test):
- products: public SELECT (anon); INSERT/UPDATE/DELETE only for seller owner or admin.
- orders: buyer can SELECT their orders; admin full access; sellers can read order_items for their products only.
- bookings: owner access for buyer or seller; admin full access.
- Ensure auth.role() mapping from Supabase JWT and use auth.uid() for ownership checks.

Serverless Functions to generate:
- POST /api/checkout/create-session
  - Creates an orders row with status = 'pending'
  - Creates order_items rows
  - Creates Stripe Checkout Session with metadata: order_id and service_type
  - Returns session.id to client
- POST /api/webhooks/stripe
  - Validates Stripe signature using STRIPE_WEBHOOK_SECRET
  - On checkout.session.completed: update orders.status = 'paid', set stripe_payment_id, create booking if service_type present, notify seller via email
- Signed upload endpoints for Supabase Storage (server-signed upload URLs)
- Admin endpoints (server-only) for seller verification, manual refunds (no service key in client)

Stripe integration:
- Use Stripe test mode for staging
- Ensure Checkout Session sets metadata.order_id and metadata.service_type
- Webhook endpoint path: /api/webhooks/stripe (add to Stripe dashboard or use Stripe CLI for testing)

CI and Deploy:

- Environments: ifa-staging and ifa-prod (separate Supabase projects or schemas and Stripe keys)
