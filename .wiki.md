# Project Summary
The Ifa Divinity Marketplace is an innovative online platform that connects Ifa divination practitioners with clients seeking spiritual readings and services. It enhances user experience through intuitive browsing, booking capabilities, and real-time data access via Supabase, while celebrating Yoruba traditions. Key features include an **Awo Dashboard**, **Consultation Workspace**, **Ebo Workflow**, **Consultation History**, **Notes + Summary**, an advanced **Scheduling Module**, a **Payments Module**, and a **Client Portal** that allows clients to manage their consultations, bookings, and payments. The **Client Booking Module** enables clients to view availability, create booking requests, and manage their booking statuses.

# Project Module Description
The project consists of the following main modules:
- **Frontend**: A React-based interface for users to explore services, manage bookings, and authenticate accounts utilizing Supabase for real-time updates. It includes functionalities for Egbo services, the Awo Dashboard, Consultation Workspace, Notes + Summary, Consultation History, Scheduling Module, Client Management module, Payments module, House Dashboard, and the new Client Portal and Client Booking Module.
- **Backend**: Manages database interactions and business logic using Supabase, handling user data and session states efficiently. It supports Egbo service management, Stripe payment processing, and APIs for the Awo Dashboard, Ebo Workflow, Consultation History, Scheduling operations, Client Management, Payments, House Dashboard, and Client Portal functionalities.

# File Description Inventory
- **README.md**: Documentation for both backend and frontend modules.
- **skills_docs/**: Documentation for Supabase integration and requirements.
- **src/**: Contains all source code for the frontend application, including components, pages, hooks, and contexts.
- **lib/audit.ts**: Implements audit logging functionality for tracking significant actions across the marketplace.
- **lib/supabase.ts**: Configures and exports the Supabase client with centralized table name mappings, now including `availability_blocks`, `availability_exceptions`, `booking_requests`, `clients`, `client_notes`, and `payments`.
- **contexts/AuthContext.tsx**: Manages user authentication and session states with real Supabase data, now includes a timeout for user role fetching to prevent indefinite loading.
- **pages/AwoClients.tsx**: Page for managing clients, allowing Awo practitioners to view, add, edit, and manage client notes and consultation history.
- **pages/AwoPayments.tsx**: Page for managing payments, including consultation and Ebo payments with functionalities for creating payments, generating payment links, and processing refunds.
- **pages/ConsultationWorkspace.tsx**: Provides Awo practitioners with a workspace for manual Odu entry, now integrated with the Notes + Summary module.
- **pages/ConsultationHistory.tsx**: Page for Awo practitioners to view past consultations, including detailed views and filtering options.
- **pages/Admin.tsx**: Admin panel for managing users, orders, products, and seller verification.
- **pages/SellerDashboard.tsx**: Dashboard for sellers to manage their products and orders.
- **pages/Cart.tsx**: Shopping cart interface with checkout functionality, including Egbo service integration.
- **pages/SuperAdminSetup.tsx**: Page for creating or elevating accounts to Super Admin, now with improved error handling and detailed error messages.
- **pages/Bookings.tsx**: Page for Super Admins to book consultations, now using a dedicated edge function for inserting bookings to bypass RLS issues.
- **pages/AwoScheduling.tsx**: Enhanced page for managing schedules, including time zone handling, drag-and-drop calendar interactions, and a calendar interface.
- **pages/HouseDashboard.tsx**: New page for managing house-level features, including practitioners, clients, consultations, shared records, and subscription management.
- **pages/ClientAuth.tsx**: New page for client authentication using magic links with registration functionality.
- **pages/ClientDashboard.tsx**: New page for clients to view dashboard summaries, including upcoming consultations, pending bookings, and quick actions.
- **pages/ClientAuthCallback.tsx**: New page for handling magic link authentication callbacks.
- **pages/ClientBookings.tsx**: New page for clients to manage booking requests and view availability.
- **pages/ClientPayments.tsx**: Updated page for clients to manage payment history and statuses, now with enhanced loading states and demo data display if authentication is delayed.
- **supabase/functions/app_notes_summary/index.ts**: Handles API endpoints for Notes + Summary operations related to consultation documentation.
- **supabase/functions/app_consultation_history/index.ts**: Handles API endpoints for Consultation History operations related to viewing past consultations.
- **supabase/functions/app_awo_clients/index.ts**: Handles API endpoints for client management operations, including client creation, updates, and consultation history retrieval.
- **supabase/functions/app_awo_payments/index.ts**: Handles API endpoints for payment operations, including transaction history, payment creation, and refunds.
- **e2e/egbo-flow.spec.ts**: Playwright E2E test for validating the complete Egbo service flow.

# Technology Stack
- **Frontend**: React, TypeScript, Tailwind CSS, Vite, FullCalendar
- **Backend**: Supabase (PostgreSQL, Authentication, Edge Functions)
- **Payment Integration**: Stripe
- **Development Tools**: ESLint, PostCSS, Playwright

# Usage
1. Install dependencies:
   ```
   pnpm install
   ```
2. Build the project:
   ```
   pnpm run build
   ```
3. Run the application:
   ```
   pnpm run start
   ```
