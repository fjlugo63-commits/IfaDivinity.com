# Project Summary
The Ifa Divinity Marketplace is an online platform that connects Ifa divination practitioners with clients seeking spiritual readings and services. It enhances user experience through intuitive browsing, booking capabilities, and real-time data access via Supabase, while celebrating Yoruba traditions. Key features include an **Awo Dashboard**, a **Consultation Workspace**, an **Ebo Workflow** module, a **Consultation History module**, and a **Notes + Summary module** for improved consultation documentation.

# Project Module Description
The project consists of the following main modules:
- **Frontend**: A React-based interface for users to explore services, manage bookings, and authenticate accounts. It utilizes Supabase for real-time updates and includes functionalities for Egbo services, the Awo Dashboard, the Consultation Workspace, the Notes + Summary module, and the Consultation History module.
- **Backend**: Manages database interactions and business logic using Supabase, handling user data and session states efficiently. It supports Egbo service management, Stripe payment processing, and APIs for the Awo Dashboard, Ebo Workflow, and Consultation History.

# File Description Inventory
- **README.md**: Documentation for both backend and frontend modules.
- **skills_docs/**: Documentation for Supabase integration and requirements.
- **src/**: Contains all source code for the frontend application, including components, pages, hooks, and contexts.
- **lib/audit.ts**: Implements audit logging functionality for tracking significant actions across the marketplace.
- **lib/supabase.ts**: Configures and exports the Supabase client with centralized table name mappings, now including the `consultation_notes`, `consultation_summary`, and `consultation_history` tables for the new modules.
- **contexts/AuthContext.tsx**: Manages user authentication and session states with real Supabase data.
- **pages/ConsultationWorkspace.tsx**: Updated page providing Awo practitioners with a workspace for manual Odu entry, now integrated with the Notes + Summary module.
- **pages/ConsultationHistory.tsx**: New page for Awo practitioners to view past consultations, including detailed views and filtering options.
- **pages/Admin.tsx**: Admin panel for managing users, orders, products, and seller verification.
- **pages/SellerDashboard.tsx**: Dashboard for sellers to manage their products and orders.
- **pages/Cart.tsx**: Shopping cart interface with checkout functionality, including Egbo service integration.
- **pages/SuperAdminSetup.tsx**: Updated page for creating or elevating accounts to Super Admin, now with improved error handling and detailed error messages.
- **pages/Bookings.tsx**: Updated page to allow Super Admins to book consultations, including fixes for role-based access and handling demo practitioner IDs.
- **supabase/functions/app_notes_summary/index.ts**: Handles API endpoints for Notes + Summary operations related to consultation documentation.
- **supabase/functions/app_consultation_history/index.ts**: Handles API endpoints for Consultation History operations related to viewing past consultations.
- **e2e/egbo-flow.spec.ts**: Playwright E2E test for validating the complete Egbo service flow.

# Technology Stack
- **Frontend**: React, TypeScript, Tailwind CSS, Vite
- **Backend**: Supabase (PostgreSQL, Authentication, Edge Functions)
- **Payment Integration**: Stripe
- **Development Tools**: ESLint, PostCSS, Playwright

# Usage
1. Install dependencies:
   ```
   pnpm install
   ```
2. Build the project:
   ```
   pnpm run build
   ```
3. Run the application:
   ```
   pnpm run start
   ```
