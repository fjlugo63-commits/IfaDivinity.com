# Project Summary
The Ifa Divinity Marketplace is an innovative online platform designed to connect Ifa divination practitioners (Awos) with clients seeking spiritual insights and services. This platform enhances user experience while honoring Yoruba cultural traditions. Key features include an **Awo Dashboard**, **Consultation Workspace**, **Ebo Workflow**, and modules for **Scheduling**, **Payments**, **Client Management**, and **Messaging**. The platform now also features a **Public-Facing Opele Consultation** interface, allowing users to engage in divination without requiring an account, enhancing accessibility and user engagement.

# Project Module Description
The project consists of the following main modules:
- **Frontend**: Developed with React, this module allows users to explore services, manage bookings, and authenticate accounts with Supabase. Key components include the Awo Dashboard, Consultation Workspace, Client Management, Payments, Messaging, and an Admin Portal. Recent updates include a `ConsultProvider` for improved state management, new rendering modules for house-specific tradition naming and cast tables, and a **UI Integration Test Harness** for testing the consultation pipeline. A new **Public-Facing Opele Consultation** interface has also been added, featuring a three-step flow: Landing Page, Consultation Form, and Result Page.
- **Backend**: Built on Supabase, it manages database operations and business logic, handling user data, session states, and payment processing via Stripe. It includes edge functions for managing system test accounts with enhanced error handling. New tables have been added to support consultation sessions, payments, and rule governance logic modules. A unified logging pipeline has been integrated for consultation enrichment and auditing.

# File Description Inventory
- **README.md**: Documentation for both backend and frontend modules.
- **skills_docs/**: Documentation for Supabase integration and requirements.
- **src/**: Contains all source code for the frontend application, including components, pages, hooks, and contexts.
- **lib/audit.ts**: Implements audit logging functionality for tracking significant actions across the marketplace.
- **lib/consultEngine.ts**: Utility for orchestrating client consultation sessions, evaluating readiness across various phases.
- **lib/consultState.ts**: Utility for managing consultation states, including type definitions and functions to navigate through states.
- **lib/supabase.ts**: Configures and exports the Supabase client with centralized table name mappings.
- **lib/opeleCastMapping.ts**: Module for applying awo-specific consecration mapping to raw Opele bits.
- **lib/patternKeyGeneration.ts**: Converts mapped bits into a pattern key and index.
- **lib/oduLookup.ts**: Looks up Odu interpretations based on house rules and active configurations.
- **lib/consultationRecordWriter.ts**: Inserts complete consultation records into the database.
- **lib/ruleProposalCreate.ts**: Creates a new rule change proposal with "pending" status and logs the action to the audit trail.
- **lib/ruleProposalReview.ts**: Moderators mark a proposal as "reviewed" with audit logging.
- **lib/ruleProposalApprove.ts**: Awo council approves a proposal, creating a new rule version and logging the action.
- **lib/ruleProposalReject.ts**: Rejects a proposal with a reason and logs the action.
- **lib/activateRuleVersion.ts**: Sets a specific rule version as the active version in consultation engine configuration.
- **lib/consultationEnrichment.ts**: Enriches consultation records with tradition-specific Odu names based on house profiles.
- **lib/consultationAuditLog.ts**: Creates audit log entries capturing consultation details.
- **lib/consultationIndexer.ts**: Generates searchable index metadata for analytics and certification.
- **lib/consultationLoggingPipeline.ts**: Unified orchestrator for the consultation logging workflow.
- **lib/renderOduName.ts**: Renders the Odu name according to house tradition and spelling maps.
- **lib/renderCastTables.ts**: Renders Ire/Osogbo, subtype, and Orisha owner according to house-specific cast tables.
- **lib/renderFullConsultation.ts**: Orchestrates the rendering of full consultation output using house traditions and cast tables.
- **contexts/AuthContext.tsx**: Manages user authentication and session states with real Supabase data.
- **contexts/consultContext.tsx**: Context provider for managing consultation state; includes `ConsultProvider` and `useConsult` hook.
- **pages/AwoClients.tsx**: Page for managing clients, allowing Awo practitioners to view, add, edit, and manage client notes and consultation history.
- **pages/AwoDashboard.tsx**: Dashboard for Awo practitioners with enhanced features including a messaging link.
- **pages/AwoMessages.tsx**: Page for Awo practitioners to manage messages with clients.
- **pages/AwoPayments.tsx**: Page for managing payments, including consultation and Ebo payments.
- **pages/AwoConsultations.tsx**: Page showing assigned consultations with status badges and navigation to the consultation workspace.
- **pages/AdminDashboard.tsx**: Admin dashboard displaying metrics and recent activities.
- **pages/AdminAwos.tsx**: Management page for Awos, including activation and house assignment.
- **pages/AdminClients.tsx**: Management page for clients, allowing for detail viewing and test account flagging.
- **pages/AdminConsultations.tsx**: Management page for consultations, including filtering by status and detail viewing.
- **pages/AdminPayments.tsx**: Management page for payments, with filtering and detail viewing.
- **pages/AdminHousesBotanica.tsx**: Management page for houses and botanica products, including product creation and order tracking.
- **pages/AdminSettings.tsx**: System settings page displaying environment and subscription info, now includes a link to the System Test Accounts page.
- **pages/SystemTestAccounts.tsx**: Page for creating and managing system test accounts, with improved error reporting showing detailed failure reasons.
- **pages/consult/history.tsx**: New page displaying consultation history with a list of sessions, including client name, state, and date, with links to view summaries.
- **pages/consult/**: Contains pages for the consultation flow:
  - **intake.tsx**: Updated client intake form for consultation, featuring fields for client name, consult reason, modality, symptoms/context, and notes.
  - **session.tsx**: Now includes a "Next Phase" button next to the phase badge in the session header to advance the consultation through its phases.
  - **summary.tsx**: Page displaying consultation summary with components for details and actions.
  - **payment.tsx**: Updated component for handling consultation payments, now inserting payment records into Supabase.
- **pages/admin/AdminTestHarness.tsx**: A new page for the **UI Integration Test Harness** featuring four tabs: Test Harness Dashboard, Record Viewer, Audit Viewer, and Rule Version Viewer.
- **pages/PublicConsultation.tsx**: New page for the **Public-Facing Opele Consultation** interface, allowing users to navigate through the consultation process without authentication.
- **supabase/functions/app_seed_test_users/index.ts**: Edge function for creating and managing test users, updated to improve error handling and add REST API fallback.

# Technology Stack
- **Frontend**: React, TypeScript, Tailwind CSS, Vite, FullCalendar
- **Backend**: Supabase (PostgreSQL, Authentication, Edge Functions)
- **Payment Integration**: Stripe
- **Development Tools**: ESLint, PostCSS, Playwright

# Usage
1. Install dependencies:
   ```
   pnpm install
   ```
2. Build the project:
   ```
   pnpm run build
   ```
3. Run the application:
   ```
   pnpm run start
   ```
