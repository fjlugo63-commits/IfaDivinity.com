# Project Summary
The Ifa Divinity Marketplace is an innovative online platform that connects Ifa divination practitioners with clients seeking spiritual insights and services. It offers a user-friendly interface focused on enhancing user experience while honoring Yoruba cultural traditions. Key features include an **Awo Dashboard**, **Consultation Workspace**, **Ebo Workflow**, **Consultation History**, **Notes + Summary**, an advanced **Scheduling Module**, a **Payments Module**, a **Client Portal**, a **Messaging Module** for real-time communication, and a **Profile Management Module**. Recent updates have improved navigation and integrated a shared layout for client pages.

# Project Module Description
The project consists of the following main modules:
- **Frontend**: Built with React, this interface enables users to explore services, manage bookings, and authenticate accounts with Supabase. It includes features like the Awo Dashboard, Consultation Workspace, Client Management, Payments, and Messaging.
- **Backend**: Utilizes Supabase for database interactions and business logic, managing user data, session states, and supporting payment processing through Stripe.

# File Description Inventory
- **README.md**: Documentation for both backend and frontend modules.
- **skills_docs/**: Documentation for Supabase integration and requirements.
- **src/**: Contains all source code for the frontend application, including components, pages, hooks, and contexts.
- **lib/audit.ts**: Implements audit logging functionality for tracking significant actions across the marketplace.
- **lib/supabase.ts**: Configures and exports the Supabase client with centralized table name mappings.
- **contexts/AuthContext.tsx**: Manages user authentication and session states with real Supabase data.
- **pages/AwoClients.tsx**: Page for managing clients, allowing Awo practitioners to view, add, edit, and manage client notes and consultation history.
- **pages/AwoDashboard.tsx**: Dashboard for Awo practitioners with enhanced features including a messaging link.
- **pages/AwoMessages.tsx**: Page for Awo practitioners to manage messages with clients.
- **pages/AwoPayments.tsx**: Page for managing payments, including consultation and Ebo payments.
- **pages/ClientAuth.tsx**: Page for client authentication using magic links.
- **pages/ClientDashboard.tsx**: Page for clients to view dashboard summaries.
- **pages/ClientMessages.tsx**: Page for clients to manage messages with their Awo advisors.
- **pages/ClientPayments.tsx**: Page for clients to manage payment history.
- **pages/ClientProfile.tsx**: Page for clients to manage their personal information.
- **pages/ClientBookings.tsx**: Page for clients to view and manage their bookings.
- **pages/ClientConsultations.tsx**: Page for clients to view their consultation history.
- **supabase/functions/app_messages/index.ts**: Handles API endpoints for messaging operations.
- **supabase/functions/app_egbo_admin/index.ts**: New functionality for managing Egbo-related administrative tasks.
- **e2e/egbo-flow.spec.ts**: Playwright E2E test for validating the complete Egbo service flow.

# Technology Stack
- **Frontend**: React, TypeScript, Tailwind CSS, Vite, FullCalendar
- **Backend**: Supabase (PostgreSQL, Authentication, Edge Functions)
- **Payment Integration**: Stripe
- **Development Tools**: ESLint, PostCSS, Playwright

# Usage
1. Install dependencies:
   ```
   pnpm install
   ```
2. Build the project:
   ```
   pnpm run build
   ```
3. Run the application:
   ```
   pnpm run start
   ```
