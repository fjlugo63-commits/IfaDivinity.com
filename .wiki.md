# Project Summary
The Ifa Divinity Marketplace is a cutting-edge online platform designed to connect Ifa divination practitioners with clients seeking spiritual insights and services. It prioritizes user experience through an intuitive interface and comprehensive booking functionalities, all while respecting Yoruba cultural traditions. Key features include an **Awo Dashboard**, **Consultation Workspace**, **Ebo Workflow**, **Consultation History**, **Notes + Summary**, an advanced **Scheduling Module**, a **Payments Module**, a **Client Portal**, a **Messaging Module** for real-time communication, and a **Profile Management Module** for secure management of client information. Recent enhancements have focused on improving navigation and integrating a shared layout for client pages.

# Project Module Description
The project consists of the following main modules:
- **Frontend**: A React-based interface that allows users to explore services, manage bookings, and authenticate accounts with Supabase for real-time updates. It includes functionalities for Egbo services, the Awo Dashboard, Consultation Workspace, Notes + Summary, Consultation History, Scheduling Module, Client Management, Payments, House Dashboard, Client Portal, Client Booking Module, Messaging Module, and Profile Management.
- **Backend**: Manages database interactions and business logic using Supabase, overseeing user data and session states efficiently. It supports Egbo service management, Stripe payment processing, and APIs for the Awo Dashboard, Ebo Workflow, Consultation History, Scheduling operations, Client Management, Payments, House Dashboard, Client Portal, Messaging operations, and Profile Management.

# File Description Inventory
- **README.md**: Documentation for both backend and frontend modules.
- **skills_docs/**: Documentation for Supabase integration and requirements.
- **src/**: Contains all source code for the frontend application, including components, pages, hooks, and contexts.
- **lib/audit.ts**: Implements audit logging functionality for tracking significant actions across the marketplace.
- **lib/supabase.ts**: Configures and exports the Supabase client with centralized table name mappings, including `availability_blocks`, `availability_exceptions`, `booking_requests`, `clients`, `client_notes`, `payments`, and `messages`.
- **contexts/AuthContext.tsx**: Manages user authentication and session states with real Supabase data, includes a timeout for user role fetching to prevent indefinite loading and error handling improvements.
- **pages/AwoClients.tsx**: Page for managing clients, allowing Awo practitioners to view, add, edit, and manage client notes and consultation history.
- **pages/AwoDashboard.tsx**: Dashboard for Awo practitioners with enhanced features including a messaging link.
- **pages/AwoMessages.tsx**: New page for Awo practitioners to manage messages with clients.
- **pages/AwoPayments.tsx**: Page for managing payments, including consultation and Ebo payments with functionalities for creating payments, generating payment links, and processing refunds.
- **pages/ClientAuth.tsx**: New page for client authentication using magic links with registration functionality.
- **pages/ClientDashboard.tsx**: Enhanced page for clients to view dashboard summaries, including upcoming consultations, pending bookings, and quick actions, now with a link to messages and profile management.
- **pages/ClientMessages.tsx**: New page for clients to manage messages with their assigned Awo advisors.
- **pages/ClientPayments.tsx**: Updated page for clients to manage payment history and statuses, now with enhanced loading states and demo data display if authentication is delayed.
- **pages/ClientProfile.tsx**: New page for clients to manage their personal information, including inline editing for name, phone, timezone, and email.
- **pages/ClientBookings.tsx**: New page for clients to view and manage their bookings.
- **pages/ClientConsultations.tsx**: New page for clients to view their consultation history.
- **supabase/functions/app_messages/index.ts**: Handles API endpoints for messaging operations including sending and retrieving messages.
- **e2e/egbo-flow.spec.ts**: Playwright E2E test for validating the complete Egbo service flow.

# Technology Stack
- **Frontend**: React, TypeScript, Tailwind CSS, Vite, FullCalendar
- **Backend**: Supabase (PostgreSQL, Authentication, Edge Functions)
- **Payment Integration**: Stripe
- **Development Tools**: ESLint, PostCSS, Playwright

# Usage
1. Install dependencies:
   ```
   pnpm install
   ```
2. Build the project:
   ```
   pnpm run build
   ```
3. Run the application:
   ```
   pnpm run start
   ```
